# 🌲 林间灵物信使 - 信件收集系统使用指南

## 📦 系统概述

信件收集系统是一个完整的2D收集玩法系统，包含以下核心功能：

- ✅ **多种信件类型** - 普通、稀有、史诗、传说、任务 5种类型
- ✅ **自动拾取逻辑** - 玩家靠近自动收集
- ✅ **背包数据存储** - 自动保存收集进度
- ✅ **实时统计UI** - 显示收集数量和分数
- ✅ **浮动动画效果** - 信件上下浮动
- ✅ **收集动画反馈** - UI脉冲动画
- ✅ **信件生成器** - 批量随机生成信件
- ✅ **详细调试日志** - 完整的执行流程日志

## 🚀 快速开始

### 3步配置好整个系统

#### 步骤1：添加核心管理器

在场景中创建空物体，命名为"Managers"，添加以下组件：

```csharp
CollectionManager  // 收集管理器（会自动创建）
```

> 💡 系统会自动创建所需组件，不需要手动配置！

#### 步骤2：添加信件生成器

在场景中创建空物体，命名为"LetterSpawner"，添加组件：

```csharp
LetterSpawner  // 信件生成器
```

在Inspector中设置：
- Letter Count: 10（生成数量）
- Spawn Radius: 15（生成范围）
- Use Different Colors: ✓（使用不同颜色）

#### 步骤3：添加收集UI

在场景中创建空物体，命名为"CollectionUI"，添加组件：

```csharp
CollectionUI  // 收集统计UI（会自动创建UI）
```

设置参数：
- Auto Create UI: ✓
- Use Collect Animation: ✓
- Show Score: ✓

完成！运行游戏即可测试收集系统。

## 📝 详细配置

### 1. LetterData 信件数据配置

#### 创建新信件

1. 右键 Project 窗口 → **Create → Collectibles → Letter Data**
2. 命名为"Letter_Common_001"
3. 在Inspector中配置：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **Letter Id** | 信件唯一ID | letter_001（自动生成） |
| **Letter Name** | 显示名称 | "森林的问候" |
| **Letter Type** | 信件类型 | Common |
| **Description** | 描述文字 | "来自森林精灵的祝福..." |
| **Letter Icon** | 图标精灵 | 拖入你的信件图标 |
| **Letter Color** | 信件颜色 | 白色/金色 |
| **Float Speed** | 浮动速度 | 1.0 |
| **Float Height** | 浮动高度 | 0.3 |
| **Score Value** | 收集分数 | 10 |
| **Can Collect Multiple** | 可重复收集 | ✗ |
| **Auto Pickup Range** | 自动拾取范围 | 1.5 |

#### 预设信件类型

| 类型 | 颜色建议 | 分数建议 | 说明 |
|------|----------|----------|------|
| **Common** | 米白 #FFF5E6 | 10分 | 普通信件 |
| **Rare** | 蓝色 #B3D4FF | 25分 | 稀有信件 |
| **Epic** | 紫色 #D9B3FF | 50分 | 史诗信件 |
| **Legendary** | 金色 #FFD700 | 100分 | 传说信件 |
| **Quest** | 绿色 #B3FFB3 | 0分 | 任务信件 |

### 2. LetterPickup 拾取逻辑

#### 信件预制件配置

创建信件预制件，添加以下组件：

```
1. SpriteRenderer    // 显示信件图标
2. CircleCollider2D  // 触发拾取
   - Is Trigger: ✓
   - Radius: 0.5
3. LetterPickup      // 拾取逻辑
```

#### LetterPickup 参数说明

| 参数 | 说明 |
|------|------|
| **Letter Data** | 拖入LetterData配置文件 |
| **Use Custom Settings** | 使用自定义覆盖配置 |
| **Use Bob Animation** | 启用上下浮动动画 |
| **Use Rotation** | 启用旋转动画 |
| **Auto Pickup** | 自动拾取（玩家进入范围自动收集） |
| **Pickup Particle Prefab** | 收集特效（可选） |

### 3. CollectionManager 收集管理器

#### 核心功能

- 管理所有已收集的信件
- 自动保存/加载进度
- 统计收集数量和分数
- 提供收集事件回调

#### 事件系统

```csharp
// 信件被收集时触发
CollectionManager.Instance.OnLetterCollected += (letter, pickup) =>
{
    Debug.Log($"收集到：{letter.letterName}");
};

// 收集数据更新时触发
CollectionManager.Instance.OnCollectionUpdated += () =>
{
    Debug.Log($"总数：{CollectionManager.Instance.TotalLettersCollected}");
};
```

#### 常用API

```csharp
// 检查是否已收集某信件
bool hasCollected = CollectionManager.Instance.HasCollectedLetter("letter_001");

// 获取收集数量
int count = CollectionManager.Instance.GetLetterCollectCount("letter_001");

// 获取某类型信件数量
int rareCount = CollectionManager.Instance.GetLetterTypeCount(LetterType.Rare);

// 重置所有进度
CollectionManager.Instance.ResetCollection();
```

### 4. CollectionUI 收集统计界面

#### 自动创建UI

系统会自动创建完整的UI界面，包含：
- 背景面板（半透明深棕色）
- 信件图标
- 收集数量文本
- 分数显示

#### 自定义UI

如果你想使用自己的UI：

1. 创建Canvas和面板
2. 添加Text组件用于显示数量
3. （可选）添加Image作为图标
4. （可选）添加Text显示分数
5. 将组件引用拖入CollectionUI

### 5. LetterSpawner 信件生成器

#### 功能特点

- 在指定范围内随机生成信件
- 可配置信件池，随机选择信件类型
- 支持自定义颜色
- 一键清除所有生成的信件

#### 使用方法

1. 在场景中添加LetterSpawner组件
2. 配置生成参数
3. 右键组件 → **生成信件**（编辑模式）
4. 或勾选 Spawn On Start（运行时自动生成）

## 🔍 调试与排错

### 查看控制台日志

系统会输出详细的执行日志，搜索关键词：
- `【信件系统】` - 系统运行日志
- `【收集管理器】` - 数据管理日志

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| **信件不显示** | 没有SpriteRenderer或图标 | 检查SpriteRenderer组件 |
| **无法拾取** | 玩家没有Collider2D | 给玩家添加碰撞体 |
| **拾取没反应** | 没有设置LetterData | 配置LetterData文件 |
| **UI不显示** | 没有Canvas | 系统会自动创建，检查Console |
| **进度没保存** | Save Progress关闭 | 开启CollectionManager的保存 |
| **没有自动收集** | Auto Pickup关闭 | 勾选LetterPickup的自动拾取 |

### 调试技巧

1. 查看Gizmos：选中LetterPickup查看拾取范围（黄色圆圈）
2. 选中LetterSpawner查看生成范围（青色圆圈）
3. 在Console窗口过滤"信件系统"查看所有相关日志

## 🔧 高级功能

### 集成对话系统

可以在收集特定信件后触发对话：

```csharp
void Start()
{
    CollectionManager.Instance.OnLetterCollected += OnLetterCollected;
}

void OnLetterCollected(LetterData letter, LetterPickup pickup)
{
    if (letter.letterId == "letter_special_001")
    {
        // 触发特殊对话
        DialogueManager.Instance.StartDialogue(specialDialogue);
    }
}
```

### 自定义收集条件

继承LetterPickup重写拾取逻辑：

```csharp
public class QuestLetterPickup : LetterPickup
{
    public bool requiresQuestActive = true;

    private void OnTriggerEnter2D(Collider2D other)
    {
        if (requiresQuestActive && !QuestManager.IsQuestActive)
        {
            return; // 任务未激活不能收集
        }
        base.OnTriggerEnter2D(other);
    }
}
```

### 信件分类背包

可以扩展CollectionManager实现分类显示：

```csharp
public List<CollectedLetter> GetLettersByType(LetterType type)
{
    return collectedLetters.FindAll(l => l.letterType == type);
}
```

## 💡 最佳实践

### 1. 信件数量建议

- 普通信件：30-50封
- 稀有信件：10-15封
- 史诗信件：5-8封
- 传说信件：2-3封
- 任务信件：根据任务数量

### 2. 生成策略

- 将生成器放置在森林不同区域
- 稀有信件放在更难到达的位置
- 任务信件放在NPC附近

### 3. 性能优化

- 场景中同时存在的信件不超过50封
- 离玩家远的信件可以暂时禁用
- 使用对象池复用信件对象
